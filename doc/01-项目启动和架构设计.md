# 项目启动和架构设计

## 项目概述

**项目名称**：Kid Climber - 攀爬架结构设计软件

**项目目标**：开发一款能在PC上离线运行、支持设计攀爬架造型的软件

**目标平台**：Windows 和 Mac 系统

**技术栈**：
- 前端：React + Vite + TypeScript + Three.js
- 后端：Go + Gin + GORM + SQLite
- 桌面打包：Tauri

## 需求分析

### 核心功能需求

1. **造型设计**
   - 3D空间设计，支持旋转查看各种角度
   - 拖拽式操作，支持组件放置、旋转、移动
   - 预设组件库，包含管子、接头、平台、附件等
   - 支持导出设计文件

2. **资源预估**
   - 用户可输入已有零件数量
   - 计算材料需求和短缺
   - 考虑连接损耗
   - 生成可打印的材料清单

3. **规格参数**
   - 支持查看管子规格（30cm和15cm）
   - 显示组件尺寸和连接信息

### 组件类型

**基础管件**：
- 30cm直管
- 15cm直管

**连接件**：
- 90度弯头
- 45度弯头
- 三通接头
- 四通接头

**平台类**：
- 小平台（30x30cm）
- 中平台（60x60cm）
- 大平台（90x90cm）

**附件**：
- 秋千
- 滑梯
- 绳梯

### 交互需求

- 3D视图控制（旋转、缩放、平移）
- 组件拖拽放置
- 组件旋转和移动
- 连接点自动吸附
- 撤销/重做功能
- 复制/粘贴功能

## 技术架构设计

### 整体架构

```
┌─────────────────────────────────────────────────┐
│                桌面应用 (Tauri)                   │
├─────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌──────────┐ │
│  │  React前端   │  │  Go后端服务  │  │  SQLite  │ │
│  │  Three.js   │←→│  业务逻辑   │←→│  数据库   │ │
│  │  状态管理    │  │  资源计算   │  │          │ │
│  └─────────────┘  └─────────────┘  └──────────┘ │
└─────────────────────────────────────────────────┘
```

### 技术选型

**前端技术栈**：
- React 18：UI框架
- TypeScript：类型安全
- Vite：构建工具
- Three.js：3D渲染引擎
- react-three-fiber：React集成
- Zustand：状态管理
- Ant Design：UI组件库
- CSS Modules：样式方案

**后端技术栈**：
- Go 1.21+：编程语言
- Gin：Web框架
- GORM：ORM框架
- SQLite：嵌入式数据库

**桌面打包**：
- Tauri 2.0：轻量级桌面应用框架

### 数据流设计

```
用户操作 → 前端状态更新 → 3D场景渲染
    ↓
保存操作 → REST API → Go后端 → SQLite
    ↓
加载操作 → REST API → Go后端 → SQLite → 前端状态
```
